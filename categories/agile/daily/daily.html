Daily Meeting
Framework Scrum

Objectifs

Le Daily Scrum appartient à l'équipe. Utilise-le pour avancer ensemble, pas pour contrôler !

Synchroniser l'équipe

Identifier les obstacles

Adapter le Sprint Backlog (si besoin)

Favoriser la transparence

Durée

15 minutes

Fréquence

Tous les jours à la même heure

Acteurs

L'équipe de Développement Le Scrum Master et le PO peuvent participer si travaillent activement sur des éléments du Sprint Backlog 

Rôles

Equipe Développement : Peuvent choisir la structure et les techniques qu'ils souhaitent, tant que leur progression ne compromette pas l'Objectif du Sprint et un plan d'action pour la journée de travail est définit. Cela permet de se focaliser et d'améliorer l'autogestion.

Outils

Tableau physique : post-its, magnets

Tableau numérique : Jira, Trello, Miro

Timer : pour respecter la durée.

Bonnes Pratiques

Rester debout : Encourage la concision ;

Respecter le temps : 15 minutes, pas de digressions ;

Focus sur l'Objectif du Sprint : Parler des tâches du Sprint en cours ;

Problèmes complexes : Les laisser pour après le Daily ("parking lot")

Tableau Scrum/Kanban : S'appuyer visuellement sur le board

Astuces pour un daily efficace

Même heure et même lieu

Chacun s'exprime, pas de reporting (au PO et/ou Scrum Master)

Encourager la collaboration

Mettre à jour le board à temps réel

Lister les obstacles pour action immédiate

Anti-Patterns à éviter

Faire du Daily un reporting au chef ;

Dépasser les 15 minutes ;

Résoudre les problèmes techniques pendant le Daily ;

Laisser certains membres monopoliser la parole ;
